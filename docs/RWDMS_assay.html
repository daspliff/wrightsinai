Introduction

Data Collection

The data for this analysis was collected using a PowerShell script that recursively iterates through the RWDMS filesystem and outputs the directory structure and file information. The script (included below) generates a text file with the following columns: Directory, Full Path, File Name, File Extension, File Size, Last Write Time, and File Owner.

Get-ChildItem -Recurse -File | ForEach-Object { 
    $firstDir = ($_.DirectoryName -split '\\')[1] #
    $owner = (Get-Acl $_.FullName).Owner
    "$firstDir, $($_.DirectoryName), $($_.Name), $($_.Extension), $($_.Length), $($_.LastWriteTime), $owner"
} > D:\tree\file_paths_20230905.txt

Basic Structure Analysis

Lets start by looking at the number of files in each directory. Weâ€™ll use the root_dir column to group the data and then count the number of files in each group. Also, weâ€™ll remove the tree directory from the data since this was a directory we created to store the outputs generated from the powershell script. Weâ€™ll then arrange the data in descending order by the number of files in each directory and calculate the total size of each directory. Finally, weâ€™ll add two columns that calculates the fractional number of files in each directory, and the fractional size of each directory, respectively.

library(DT)
library(dplyr)
library(knitr)

total_files <- rdwms_data %>%
    filter(root_dir != "tree") %>%
    nrow() # Total number of files

rdwms_data %>%
    filter(root_dir != "tree") %>%
    group_by(root_dir) %>%
    summarise(n_files = n(), total_size = sum(file_size)) %>%
    arrange(desc(n_files)) %>%
    mutate(
        n_files = n_files, # Remove comma formatting for calculation
        size_GB = total_size / (1024^3), # Convert bytes to GB
        frac_num = n_files / total_files, # Calculate percentages
        frac_size = total_size / sum(rdwms_data$file_size, na.rm = TRUE)
    ) %>%
    select(root_dir, n_files, size_GB, frac_num, frac_size) %>%
    mutate(
        n_files = scales::comma(n_files), # Add comma formatting
        size_GB = scales::comma(round(size_GB, 2)),
        frac_num = paste0(round(frac_num * 100, 2), "%"),
        frac_size = paste0(round(frac_size * 100, 2), "%")
    ) %>%
    datatable(
        options = list(
            columnDefs = list(list(className = "dt-left", targets = 1)),
            scrollX = TRUE, # Enable horizontal scrolling

File Type Analysis

Lets take a look at the distribution of file types in the RWDMS filesystem. Weâ€™ll start by tabulating the number of files of each file type. Weâ€™ll then calculate the total space occupied by each file type by summing across the size variable for each file type. Weâ€™ll also add a column that calculates the fractional number of files in each file type, and the fractional size of each file type, respectively. This summary table is displayed below.

expand for full code 
library(DT)
rdwms_data %>%
    group_by(file_ext) %>%
    summarize(count = n(), size_GB = (sum(file_size) / (1024^3))) %>%

    mutate(
        num_frac = paste0(round(count / sum(count) * 100, 3), "%"),

Checking for Duplicates

Lets check to see if there are any files that are duplicated across multiple directories. Because we are interested in the duplication of large files, weâ€™ll filter the dataset for files greater than 100MB. Weâ€™ll start by creating a table of the number of files in each file name. Weâ€™ll then arrange the data in descending order by the number of files in each file name and calculate the total size of each file name. Weâ€™ll also add a column that calculates the fractional number of files in each file name, and the fractional size of each, respectively.

File Inventory

Lets create a searchble invetory of all the files in the RWDMS filesystem. Weâ€™ll start by creating a table of all the files in the RWDMS filesystem. We are especially interested in environmental datsets across the filesystem, so weâ€™ll filter for files with text-based (.csv, .txt, .json), binary (.nc, .ncf, .hdf, .h5), geospatial (.tif, .tiff, .shp, .shx, .dbf, .kml, .kmz, .grib, .las), database (.sql, .sqlite, .db, .gdb), and archive (.zip, .rds, RData) file extensions.

# Create a table of all files in the RWDMS filesystem
rdwms_data %>%
    filter(root_dir != "tree") %>%
    filter(
        tolower(trimws(file_ext)) %in% c(
            ".csv", ".txt", ".json",
            ".nc", ".ncf", ".hdf", ".h5",
            ".tif", ".tiff", ".shp",
            ".kml", ".kmz", ".grib", ".las",
            ".sql", ".sqlite", ".db", ".gdb"
        )
    ) %>%
    filter(file_size >= 1024^2 * 100) %>% # Filter for files >= 100MB
    arrange(file_ext) %>%
    mutate(
        file_size = round(file_size / (1024^3), 2), # Convert bytes to GB
    ) %>%
    datatable( # Use the datatable() function instead of kable() and kable_styling()
        options = list(
            scrollX = TRUE, # Enable horizontal scrolling

            style = "caption-side: bottom; text-align: center;",
            "Table 5: ", htmltools::em("Inventory of large environmental datasets in the RWDMS filesystem (files >= 100MB)")
        )
    ) %>%
    DT::formatStyle(columns = c(1:7), fontSize = "11pt")
